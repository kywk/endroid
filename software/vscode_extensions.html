[VS Code] VS Code Extensions, 2019

Posted on 2019-03-21 | Comments:

Preface

Visual Studio Code 在微軟及社群頻繁的維護與更新下, 功能愈來愈齊全強大, 社群裡的分享也愈來愈多人使用 VS Code 進行開發 Demo, 聲勢已經超越 Sublime Text 許多.

因介面習慣, 我大部份專案仍以 Sublime Text 進行開發, 而 .NET Core 專案上使用 VS Code 進行開發. 不過近來也漸漸使用 VS Code 在其他專案上.

VS Code 擴充套件實在非常多, 幾乎不同語言都有完整生態系的支援. 個人套件清單列舉如下:
這篇也許定期更新, 並加上 ChangeLog.

Settings Sync

透過 GitHub Gist同步設定檔和安裝的套件, 支援背景自動同步

Core & common

Editing

  • EditorConfig for VS Code
    EditorConfig 是個幫助團隊有統一規範的工具, 尤其遇到團隊中有人用 不同的 作業系統或編輯器時， 統一各種的不同

  • Auto Close Tag
    把右括號或結束標籤補上, VS Code 雖有內建基本補完功能, 這個套件支援更多符號和設定, 例如打 </ 的時候也幫我補上 HTML close tag。

  • Code Spell Checker
    所有程式的變數或是方法命名都應該是 有意義 的, 不是隨便的縮寫. 這個套件可以檢查英文單字是否有拼錯, 且支援駝峰命名

Look & feel

  • Prettier — Code formatter
    可以與 project 中的 prettier 設定檔互動. 設定自動 format 的時機點, 例如: 存檔就 format

  • Guides
    在程式開關符號 (例如左右大括號) 拉一條線, 方便識別程式區塊

  • Indent-Rainbow, Indenticator
    視覺化縮排, 兩個搭配可讓程式碼對齊與縮排更一目了然

  • Bracket Pair Colorizer
    程式碼區塊的對應, 用不同顏色表示 {} [] () 的程式區塊, 異於區分

  • DotENV
    讓 .env 檔也有 syntax 效果

Coding

  • Code Runner
    支持多种语言的代码的立即执行

Languages

  • C#
    微軟基本的 C# 套件
    • C# FixFormat
      強化 VS Code 對於 C# 的 format 能力
    • C# Extensions
      強化 VS Code 對於 C# 的 Coding IntelliSense 能力
  • Go VS Code 是寫 Go 目前最多人使用的 IDE, 這套可以涵蓋所有你寫 Go 時所需的功能

JavaScript, Node

  • ESLint
    可以與 project 中的 .eslintrc 互動, 讓專案中的 ESLint 規則在 Coding 的時候就提醒是否有違反 ESLint 的規則

  • Quokka.js, Wallaby.js
    即時執行 JavaScript 程式碼內容, 快速 Demo 時很方便

  • code snippets

    • JavaScript (ES6) code snippets
      就是 JS 的 Snippets。也同時支援 TypeScript、React、Vue
    • ES7 React/Redux/GraphQL/React-Native snippets
      這套涵蓋了 React/Redux/GraphQL/React-Native 的大部分 code snippets，如果你不想客製化自己的 snippets，這套會是你很好的選擇。
    • Babel JavaScript
      JavaScript 語法高亮, 支援 ES201x, React JSX, Flow 與 GraphQL

HTML5, Web Frontend

  • Vue.js
    • Vetur
      Vue 的集大成工具，寫 Vue 必裝
    • Vue 2 Snippets
      Vue 2 的程式碼片段

DevOps

Project Management

  • Project Manager
    如果同時有許多 project 要開發, 這個套件可以快速切換 project, 非常方便

  • Bookmarks
    在開發時, 用書籤標記位置, 方便後續回來找位置

  • TODO Highlight
    Highlight 程式碼中的 TODO, FIXME 註解, 還可以統整整個專案的 TODOs

SCM

  • Git Lens
    必備的全能 Git 擴充功能, 可以在閃爍游標指到哪一行他就會顯示該行的 Git Blame, 可以快速查看是哪個 commit 修改的, 追 code 很方便. 缺點是會塞很多功能在右鍵, 不喜歡的話記得設定調整

  • Better Comments
    编写更加人性化的注释

Themes

  • Material Theme, One Dark Pro
    好的色彩主題是一定要裝的

  • Material Icon Theme
    好的圖示主題也是一定要裝的
